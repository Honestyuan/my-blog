    # JSONP的缺点

    • JSON只支持get，因为script标签只能使用get请求
    • JSONP需要后端配合来返回指定格式的数据

    # 跨域(jsonp ajax)

    JSONP： ajax请求受同源策略影响，不允许进行跨域请求，而script标签的src属性中的链接却可以访问跨域的js脚本，那么利用这个特性，服务端不再返回JSON格式的数据，而是返回一段调用某个函数的js代码，在src中进行调用，这样实现了跨域

    # 如何实现跨域

    1. JSONP 通过动态创建script，再请求一个带参网址实现跨域通信
    2. document.domain+iframe跨域 两个页面都通过js强制设置document.domain为基础主域，就实现了同域。
    3. location.hash+iframe跨域 a欲与b进行跨域通信，通过中间页c来实现，三个页面不同域的利用iframe的locaton.hash传值，相同域之间直接通过js访问来通信
    4. window.name+iframe跨域 通过iframe的src属性由外域转向本地域，跨域数据即从iframe的window.name从外域传递到本地域。
    5. postMessage跨域 可以跨域操作的window属性之一
    6. CORS 服务端设置Access-Control-Allow-Origin,前端无须设置，若要带cookie请求，前后端都需设置
    7. 代理跨域 起一个代理服务器，实现数据的转发

    # DOM是什么，你的理解

    DOM是文档对象模型，是w3c组织推荐的处理可扩展标志语言的标准编程接口。在网页上，组织页面的对象被组织在一个树形结构中，用来表示文档中对象的标准模型就称为DOM

    # DOM有哪些api

    节点创建型、页面修改型、节点查询型、节点关系型、元素属性型、元素样式型

    # ajax的返回状态

    0 － （未初始化）还没有调用 send()方法
1 － （载入）已调用 send()方法，正在发送请求
2 － （载入完成）send()方法执行完成，已经接收到全部响应内容
3 － （交互）正在解析响应内容
4 － （完成）响应内容解析完成，可以在客户端调用了
未调用-调用且发送-发送完成且接收-解析接收-解析完成

    # 如何实现一个ajax

    创建一个XMLHttpRequest对象，并操作该对象

    1. 设置请求的参数请求方式 请求页面的相对路径 是否异步
    2. 使用onreadystatechange设置一个回调函数,来处理服务器响应请求
    3. 获取异步对象的readyState属性，该属性存有服务器的响应状态
    4. 判断响应报文的状态，若为200说明服务器正常允许且返回响应数据
    5. 读取响应数据，通过responseText属性来取回由服务器返回的数据
    <script>
        function testAjax(){
            var xhr= new XMLHttpRequest();
            xhr.open("get","text.php",true);
            xhr.onreadystatechange=function(){
                if(xhr.readyState==4 && xhr.status==200){
                    var resText = xhr.responseText;
                    document.getElementById("add").innerHTML=resText;
                }
            }
            xhr.send(null);
        }
    </script>

    # Rem、em

    rem： 取决于根元素字体大小

    根元素的字体大小为 16px，那么 10rem 就等同于 10*16=160px。
    1

    em： 取决于使用em单位元素的字体大小 div 的字体为 18px，设置它的宽高为 10em，那么此时宽高就是 18px*10

    1.一定要记住的是em是根据使用它的元素的font-size的大小来变化的，而不是父元素字体大小。
2.em单位存在继承的时候，每个元素将自动继承其父元素的字体大小
html 元素的字体大小虽然是直接确定 rem 的值， 但这个字体大小首先是来源于浏览器的设置
所以一定要设置 html 的值的大小，因 为有可能用户的浏览器字体大小是不一致的
html{
font-size: 1.5em;
}
可以看到，因为浏览器默认字体大小为 16px，所以当设置 HTML 的 fontsize 的值为 1.5em的售后，其对应的 px 的值为 16*1.5=24px
所以**em优缺点：**

- `em 可以让我们的页面更灵活，更健壮，比起到处写死的 px 值，em 似乎更有张力，改动父元素的字体大小，子元素会等比例变化，这一变化似乎预示了无限可能`
- `em 做弹性布局的缺点还在于牵一发而动全身，一旦某个节点的字体大小发生变化，那 么其后代元素都得重新计算`此时，再设置其他元素的 rem 的值的时候，其对应的像素值为 n*24px

    em优缺点：

    • em 可以让我们的页面更灵活，更健壮，比起到处写死的 px 值，em 似乎更有张力，改动父元素的字体大小，子元素会等比例变化，这一变化似乎预示了无限可能

    rem优缺点：

    • 可以快速适用移动端布局 字体图片 高度
    • ie的兼容性低，对pc页面来讲使用次数不多
    • 数据量打，所有内容都需要我们取给一个准确的值，才能保证不同机型适配
    • 在响应式布局中，必须通过js来动态控制根元素font-size的大小

    # flex布局优缺点

    优点： 容易上手，布局灵活，根据规则很容易达到某个布局效果

    缺点： 浏览器兼容性较差，只能兼容ie9以上

    # 百分比布局

    百分比布局有两个缺点：

    • 计算困难，得将设计稿换算成百分比单位
    • 各个属性中如果使用百分比，相对父元素的属性并不是唯一的。margin、padding 不管垂直 还是水平方向都相对比父元素的宽度、border-radius 则是相对于元素自身等等。造成我们使用百分比单位容易使布局问题变得复杂。

    # vw单位

    vw单位与百分比类似，百分比单位的换算困难，vm更像是理想的百分比的单位。任意层级元素，在使用vw单位的情况下，vw都等于视图宽度的百分之一。

    vw转换成px只需要知道视图宽度即可

    # toB和toC的区别

    首先明确toB面向企业、toC面向用户
toB 产品应更考虑获益与系统性价值，部分情况还有可能会牺牲掉局部个体的利益，对于使用者而言应该是自律或他律的
toC 产品则更考虑的是个体用户的偏好，并长时间内，基于技术效率的提升，产品的服务中心更多地围绕着更高效地帮助用户的"欲望"释放进行设计，对于使用者而言是一个释放自我的存在

    # 点击穿透

    什么是点击穿透？

    假设页面有两个元素AB。B在A元素之上，我们在B元素的touchstart事件上面注册了一个回调函数，作用就隐藏B元素。我们发现，当我们点击B元素，B元素被隐藏了，随后A元素触发了click事件

    原因

    这是因为在移动浏览器，事件的执行顺序是touchstart - touchend - click。而click有300ms的延迟，所有当touchstart事件把B元素隐藏后，隔了300ms就触发了click事件，所有该事件被派发到A元素上，如果A元素是链接，那么此时页面就会跳转
